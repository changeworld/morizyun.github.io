(クックパッド) @naoya_ito さん x @hotchpotch さんの対談

伊藤直也さん(@naoya_ito)とクックバッドCTOの舘野祐一さん(@hotchpotch)の対談記事。

  • 【前編】 元・超ワガママエンジニアのクックパッドCTOと語る、「CTOって何する人だ？」論 / 飲み会で探るエンジニアのホンネ

  • 【後編】 元・超ワガママエンジニアのクックパッドCTOと語る、「CTOって何する人だ？」論 / 飲み会で探るエンジニアのホンネ

    • 技術基盤・CTO(CIO)としての改善

      • 技術的にもっとよくできるところをよくしたりとか、ボトルネックになってるところを改修
      • 使いやすく、アクセスしやすく、かつセキュリティもしっかり担保できる情報管理のあり方を取り決める
      • テクノロジーの重要性を経営にフィードバックする(数字の根拠が無い部分を情熱で押し切る)

    • マネージメント

      • 組織全体をもっと良い方向に変える。エンジニアにとって居心地の良い組織を作る
      • 会社にとって必要な技術は当然エンジニアだったら吸収すべきというカルチャーを作る
      • 自由度は100%ではダメだけど、0もダメ。一定の自由度がENGに与えられるようにする
      • 仕事に意味がある・価値がある・認められていると感じて、やりがいを見つけられるようにする

    • 情報共有

      • はてなグループ => ブログとwikiを足して2で割ったような情報共有ツール
      • 日々やった業務やみんなに周知したいことをブログで発信したり、wikiに情報をまとめる

    • 情報発信

      • 社内外のエンジニアにどれだけ貢献するかっていう評価軸を持つ => OSSや外部発表を促す
      • 基本は楽しいから外部で発表している。その雰囲気や仕組みを作ることが大切

(クックパッド) 週1回×15分の個人面談

クックパッドエンジニアのLeonard Chinさんの「週1回×15分の個人面談」によるの事例。

  • 週1回×15分でチーム変革！事業を成功に導くクックパッドの振り返り

  • 会議体を2つもつ

    • 進捗確認・スタンドアップミーティング
      • すでに表面化している仕事のススメ方に関する進捗や課題の整理を行う
    • 個人面談
      • チームのいろいろな課題を発見する。解決に導くヒントに出会える。メンバーと強い信頼関係を作る
      • 話したいことを話す。仕事の話題でも、プライベート、技術ネタでもOK
      • 質問の例としては以下の様なものがある
        • (個人が抱えている課題を探す) 仕事のブロッカーありますか？
        • (継続的に学ぶことを促す) 最近、試してみたい技術ありますか？、最近ひどいと思ったコードはありますか？
        • (潜在的ストレスを探す) なにか不安はありますか？、いまの仕事、楽しいですか？
      • 具体的な目標設定やフォローアップは行わない
