オブジェクト指向プログラミング 初歩の初歩

Jun 22nd, 2014

  • Tweet

オブジェクト思考が生まれた歴史と、オブジェクト思考の基礎について。


構造化プログラミング

構造化プログラミング - Wikipedia

ある土台の上にさらに抽象化した土台をおき、その上にさらに・・・というようにプログラムをくみ上げていく思想のプログラムである。

プログラミングのモジュール化

プログラムをモジュール化する上で重要な考え方は次の３つ。

結合度

結合度 - Wikipedia

利用者、もしくはメンテンナンスをする者にとって対象を利用、保守しやすいかを表す尺度。コラボレーションのしやすさを表す。よい結合とは、定められたデータの受け渡し(データ結合)や、メッセージの送信(メッセージ結合)のように内部構造に依存せず、情報のやりとりが明確な状態を言う。

凝集度

凝集度 - Wikipedia

コードのまとめ方の「良さと悪さ」を表す尺度。クラスごとに責任の範囲を明確にすることで、コードの読みやすさや再利用を高めることができる。良い凝集とは、

1) 通信的凝集 : データに触れる処理をまとめること
2) 情報的凝集 : 適切な概念とデータ構造、アルゴリズムをまとめること
3) 機能的凝集 : 一つに定期されたタスクをこなせるようにまとめること

関心の分離

関心の分離 - Wikipedia

オブジェクト思考や、MVCのようなデザインパターンなどのプログラミングパラダイム(プログラミングの見方)、関心の分離を実践するための手法である。関心の分離にはモジュール性とカプセル化といった概念が重要となる。
